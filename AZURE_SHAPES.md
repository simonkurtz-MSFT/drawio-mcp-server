# Azure Architecture Icons for Draw.io MCP Server

This implementation adds support for Microsoft Azure architecture icons to the draw.io MCP server, enabling AI agents to generate technical architecture diagrams using official Azure symbols.

## Overview

The Azure shape library includes 100+ official Azure service icons organized into categories:

- **Compute** - Virtual Machines, Functions, Kubernetes, App Services, etc.
- **Storage** - Blob Storage, File Storage, Disks, Queues, Tables
- **Networking** - Virtual Networks, Load Balancers, VPN Gateways, Firewalls, etc.
- **Databases** - SQL Database, Cosmos DB, Cache for Redis, etc.
- **AI + Machine Learning** - Cognitive Services, Azure OpenAI, Bot Service
- **Analytics** - Synapse Analytics, Databricks, Event Hubs, etc.
- **Security + Identity** - Azure AD, Key Vault, Security Center, Sentinel
- **Management + Governance** - Monitor, Log Analytics, Automation, Policy
- **Integration** - Service Bus, Logic Apps, API Management, Event Grid
- **DevOps** - Azure DevOps, Pipelines, Repos
- **General** - Resource Groups, Subscriptions, Management Groups

## Usage with MCP Server

### Browsing Available Azure Shapes

```
1. List all shape categories (includes Azure categories):
   Tool: get-shape-categories

2. Get shapes in a specific Azure category:
   Tool: get-shapes-in-category
   Parameters: { "category_id": "compute" }

3. Find a specific Azure shape by name:
   Tool: get-shape-by-name
   Parameters: { "shape_name": "azure-virtual-machine" }
```

### Creating Azure Architecture Diagrams

```
1. Add an Azure service icon to the diagram:
   Tool: add-cell-of-shape
   Parameters: {
     "shape_name": "azure-virtual-machine",
     "x": 100,
     "y": 100,
     "text": "Web Server VM"
   }

2. Connect Azure services with edges:
   Tool: add-edge
   Parameters: {
     "source_id": "cell-1",
     "target_id": "cell-2",
     "text": "HTTPS"
   }

3. Export the diagram:
   Tool: export-diagram
   Result: Draw.io XML format
```

### Example: Creating a Simple Azure Architecture

```typescript
// 1. Add Azure Virtual Network
add-cell-of-shape({
  shape_name: "azure-virtual-network",
  x: 50,
  y: 50,
  width: 300,
  height: 200,
  text: "VNet"
})

// 2. Add Virtual Machine
add-cell-of-shape({
  shape_name: "azure-virtual-machine",
  x: 100,
  y: 100,
  text: "VM"
})

// 3. Add Load Balancer
add-cell-of-shape({
  shape_name: "azure-load-balancer",
  x: 250,
  y: 100,
  text: "Load Balancer"
})

// 4. Connect them
add-edge({
  source_id: "cell-2",
  target_id: "cell-3",
  text: "Traffic"
})

// 5. Export to XML
export-diagram()
```

## Azure Shape Names

All Azure shapes follow the naming convention: `azure-<service-name>`

Examples:
- `azure-virtual-machine`
- `azure-app-service`
- `azure-kubernetes-service`
- `azure-sql-database`
- `azure-cosmos-db`
- `azure-storage-account`

- `azure-active-directory`

## Important Notes

### Draw.io Azure Library Requirement

The Azure shapes generated by this MCP server **require the Azure shape library to be enabled in draw.io** when opening the diagram:

1. Open the generated `.drawio` file in [draw.io](https://app.diagrams.net/)
2. Click **More Shapes** at the bottom of the left panel
3. In the **Networking** section, enable **Azure**
4. Click **Apply**

The Azure icons will then render correctly with official Microsoft styling.

### How It Works

The MCP server generates XML that references draw.io's built-in Azure icon library using style strings like:

```
aspect=fixed;html=1;points=[];align=center;image;fontSize=12;image=img/lib/azure2/compute/Virtual_Machine.svg;
```

These references are resolved when the file is opened in draw.io with the Azure library enabled.

## Best Practices for Azure Diagrams

### Labeling

- Keep labels concise (service name or role)
- Use edge labels to indicate data flow or protocols

### Organization

- Group related services in the same layer
- Use containers (rectangles sent to back) to indicate VNets or resource groups
- Arrange services logically (e.g., web tier → app tier → data tier)

### Connections

- Use solid lines for primary data flow
- Consider using different edge styles for different types of connections

## Limitations

- Icons are references to draw.io's built-in library, not embedded SVG
- Requires internet connection to load icons when opening in draw.io
- Azure library must be manually enabled in draw.io (one-time setup)
